# 微信开发者工具缓存清理指南

## 问题现象

即使代码已修复，仍然报 `gridColumns is not defined` 错误。

## 原因分析

微信开发者工具缓存了旧版本的编译文件，即使源文件已修改，它仍在使用旧缓存。

## 解决方案

### 方法1：重启微信开发者工具（推荐）

1. **完全关闭微信开发者工具**
   - 关闭项目窗口
   - 退出微信开发者工具程序
   - 确保进程已结束

2. **等待10秒**
   - 让文件系统完成写入
   - 确保缓存已释放

3. **重新打开项目**
   - 启动微信开发者工具
   - 重新打开 `pdDraw` 项目
   - 等待项目索引完成

4. **重新编译**
   - 点击 "编译" 按钮
   - 或按 `Ctrl + B`
   - 查看控制台，确认无错误

### 方法2：手动删除缓存目录

```bash
# 删除以下目录（如果存在）
rm -rf d:/qupindou/pdDraw/unpackage/
rm -rf d:/qupindou/pdDraw/dist/
```

然后重新编译。

### 方法3：修改文件触发重新编译

1. **任意修改一个文件**
   - 例如：在 `generate.vue` 中添加一个空行
   - 保存文件

2. **立即删除修改**
   - 删除刚才添加的空行
   - 保存文件

3. **重新编译**
   - 这会强制工具重新扫描文件

### 方法4：清除工具缓存

在微信开发者工具中：
- 点击菜单 "工具" → "清除缓存" → "全部清除"
- 然后重新编译

## 验证修复

清理缓存后，检查以下位置：

```javascript
// 在 pages/generate/generate.vue 中，应能找到：

// 第268-270行附近
const pixelationMode = ref('dominant')
const gridColumns = ref(50)        // ✅ 确认存在
const colorMergeThreshold = ref(30)

// 在 processImage 函数中（第808行附近）
console.log('横向切割:', gridColumns.value)  // ✅ 确认能访问
```

## 如果问题仍然存在

1. **检查文件是否保存**
   - 确认 `generate.vue` 已保存
   - 文件行数应在 2108-2110 行左右

2. **检查变量声明位置**
   - 确保在 `<script setup>` 内
   - 确保在 `processImage` 函数之前

3. **重新克隆项目**
   - 备份修改
   - 重新拉取代码
   - 应用修改

## 预防措施

以后修改代码后：
- ✅ 养成重启工具的习惯
- ✅ 定期清理缓存
- ✅ 使用版本控制（git）

---

**重要提示**：微信开发者工具的缓存机制有时很顽固，重启工具通常是最有效的解决方案。
